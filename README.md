# Ksylian

Self-hosted панель для создания и управления Minecraft-серверами.

Ksylian помогает поднять Minecraft-сервер на своем хосте без ручной возни с папками, `server.jar`, сервисами и логами. Идея проекта простая: открыть веб-панель, выбрать тип сервера, версию Minecraft, нажать кнопку создания и дальше управлять сервером из одного места.

> Проект находится на ранней стадии разработки. Часть функций уже работает, часть еще активно проектируется.

## Зачем это нужно

Если у тебя есть домашний сервер, VPS или отдельный Linux-компьютер под Minecraft, Ksylian должен стать удобной панелью поверх этой машины:

- быстро создавать новые Minecraft-серверы;
- запускать, останавливать и перезапускать их из браузера;
- смотреть логи без SSH;
- видеть нагрузку на хост;
- редактировать базовые настройки сервера;
- управлять доступом к панели через пользователей.

Проект ориентирован на self-hosted сценарий: ты сам контролируешь сервер, данные и окружение.

## Что уже умеет

Главный фокус сейчас — развертывание и обслуживание Minecraft-серверов на одном хосте.

- Создание Minecraft-сервера через веб-интерфейс.
- Выбор типа сервера в мастере создания: Vanilla, Paper, Purpur, Fabric, Forge и NeoForge.
- Выбор версии Minecraft из списка.
- Автоматическая подготовка файлов сервера.
- Скачивание серверного ядра и loader-файлов для поддерживаемого типа.
- Регистрация сервера как `systemd`-сервиса на хосте.
- Запуск, остановка и перезапуск сервера.
- Принудительная остановка, обновление серверных файлов и rollback последнего обновления.
- Просмотр логов сервера с автообновлением.
- WebSocket-консоль, фильтры логов, скачивание лога и RCON-команды.
- Отображение статуса серверов в панели.
- Редактирование базового конфига Minecraft.
- Файловый менеджер сервера: просмотр, загрузка, скачивание, редактирование, перемещение, переименование, удаление в корзину, поиск и распаковка архивов.
- Бэкапы и восстановление: live/stopped backup, manifest, checksum, restore по областям и политики хранения.
- Управление установленными модами: сканирование JAR, metadata, зависимости, client-only warnings, установка, удаление, отключение, включение, pin и массовые операции.
- CurseForge-каталог: поиск проектов, фильтры, файлы, зависимости, установка модов и импорт modpack/server pack.
- Build manifest `ksylian-manifest.json`: история, diff, import/export и source metadata.
- Safe update для CurseForge-сборок: resolver, review diff, тестовый clone и анализ запуска.
- Импорт существующих серверов с preview, определением типа/версии/порта/mod count и созданием systemd unit.
- Управление игроками: online/history, whitelist, OP, ban/pardon, kick, сообщения и базовая LuckPerms-интеграция через RCON.
- Мониторинг нагрузки хоста: CPU/RAM/SWAP с графиками, диски, сервисы и процессы.
- Первичная настройка администратора.
- Авторизация пользователей.
- Временные ссылки-приглашения для регистрации.
- Персональные цветовые темы интерфейса.
- Обновление панели из админки или терминала через release tags.

## В разработке

- Проверка сценариев CurseForge/Safe Update на живом VPS с реальными сборками.
- Интеграция Modrinth пока пропущена и остаётся отдельным будущим этапом.
- Планировщик обслуживания: рестарты, бэкапы, RCON-команды и проверки обновлений по расписанию.
- Уведомления о падениях, нехватке ресурсов, бэкапах и обновлениях.
- Minecraft-specific мониторинг: TPS, MSPT, JVM, чанки, сущности и история метрик.
- Более гибкие роли пользователей.
- Поддержка нескольких хостов.
- Обновление Host Agent как часть полноценного release workflow.

## Как это устроено

```text
frontend/   веб-интерфейс на Vue
backend/    API, авторизация, настройки и интеграции
agent/      агент на хосте, который управляет Minecraft-серверами
deploy/     Docker Compose и примеры окружения
```

Панель состоит из трех основных частей:

- **Frontend** — интерфейс администратора.
- **Backend** — API панели, авторизация, настройки и связь с агентом.
- **Host Agent** — локальный сервис на сервере, который создает папки, запускает Minecraft, читает логи и собирает метрики.

## Быстрый старт

Скопируй пример окружения:

```bash
cp deploy/.env.example deploy/.env
```

Заполни обязательные секреты:

```bash
KSYLIAN_AGENT_TOKEN=replace-with-a-long-random-token
KSYLIAN_AUTH_SECRET=replace-with-a-long-random-secret
```

> Не коммить реальные `.env` файлы. Используй `.env.example` как шаблон, а секреты храни только на своем сервере.

Запусти панель:

```bash
docker compose --env-file deploy/.env -f deploy/docker-compose.yml up -d --build
```

По умолчанию:

- Frontend: `http://SERVER_IP:8088`
- Backend: `http://SERVER_IP:8090`
- Agent: `http://SERVER_IP:8765`

При первом открытии панель предложит создать администратора.

## Установка Host Agent

Без агента панель не сможет реально создавать и запускать Minecraft-серверы. Агент должен быть установлен на той машине, где будут находиться серверы.

```bash
cd agent
cp .env.example .env
```

В `agent/.env` укажи тот же `KSYLIAN_AGENT_TOKEN`, что и в `deploy/.env`.

Затем установи сервис:

```bash
./install-agent.sh
```

Агент запускается через `systemd` и отвечает за:

- создание папок серверов;
- скачивание и подготовку серверных файлов;
- запуск и остановку Minecraft-сервисов;
- чтение логов;
- сохранение `server.properties`;
- сбор метрик хоста;
- подготовку бэкапов.

## Обновление панели

Ksylian обновляется через release tags вида `v0.6.6`. Текущая версия хранится в [VERSION](./VERSION), а `deploy/.env` получает `KSYLIAN_BUILD_VERSION` и `KSYLIAN_BUILD_SHA` во время обновления.

Основной способ обновления — из админки:

1. Открой `Настройки`.
2. Нажми проверку обновлений.
3. Если доступен новый release tag, запусти обновление.

Backend отправит команду Host Agent, а Host Agent выполнит [deploy/scripts/update-ksylian.sh](./deploy/scripts/update-ksylian.sh). Этот же скрипт можно запускать вручную на сервере:

```bash
cd /opt/ksylian
sudo deploy/scripts/update-ksylian.sh v0.6.6
```

Если версию не передать, скрипт возьмёт последний локально доступный git tag после `git fetch --tags`:

```bash
cd /opt/ksylian
sudo deploy/scripts/update-ksylian.sh
```

Старый деплой через self-hosted GitHub Runner удалён: первоначальная разработка больше не привязана к автодеплою из GitHub Actions. Для production-обновлений используются только админка или терминальный скрипт.

## Локальная разработка

Backend:

```bash
cd backend
python3 -m venv .venv
. .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

Frontend:

```bash
cd frontend
npm install
npm run dev
```

После запуска открой Vite URL и создай первого администратора.

## Статус проекта

Ksylian пока не стоит считать стабильной production-панелью. API, структура данных и механика установки могут меняться между версиями.

Сейчас проект лучше подходит для:

- домашнего сервера;
- тестового Minecraft-хоста;
- небольшой приватной команды;
- экспериментов с self-hosted инфраструктурой.
