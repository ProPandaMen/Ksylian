# Статус разработки Ksylian

Обновлено: 2026-07-23

## Текущий фокус

Активный этап: Milestone `Ksylian 0.2 — Stable Core`.

Ближайшие приоритеты:

- Довести безопасный бэкап до требований 0.2.
- Добавить импорт существующего сервера.
- Проверить RCON и WebSocket на живом VPS.
- Сделать публичный домен и proxy настраиваемыми из панели.

## Уже сделано

- Пользователи перенесены из `users.json` в SQLite.
- Обновлен дизайн страниц авторизации, настроек, пользователей, мониторинга, серверов, страницы сервера и создания сервера.
- Цветовые темы вынесены в отдельные файлы.
- Добавлен базовый путь создания Fabric и Forge серверов на уровне agent.
- Добавлены реальные загрузчики Vanilla, Paper и Purpur на уровне agent.
- Добавлена checksum-проверка загрузчиков: Vanilla через SHA1, Paper через SHA256, Purpur через MD5.
- Paper и Purpur снова доступны как отдельные типы при создании сервера.
- Agent требует `KSYLIAN_AGENT_TOKEN` и отказывается стартовать без токена.
- Agent получил базовый rate limit и журнал действий.
- Управляемые серверы создаются только внутри `KSYLIAN_SERVER_ROOT`.
- Управляемые Minecraft-серверы запускаются от отдельного системного пользователя.
- Отдельный пользователь на каждый сервер рассмотрен; текущий безопасный минимум — общий `ksylian-minecraft` с systemd-ограничениями.
- Для managed systemd-сервисов добавлены базовые security options.
- Создание managed-сервера стало атомарным: при ошибке агент очищает папку и unit.
- Создание сервера переведено в фоновую установку со статусом `INSTALLING`.
- Добавлены более подробные состояния сервера: выключен, запускается, работает, останавливается, упал.
- Добавлено API-действие принудительной остановки сервера.
- Добавлено действие обновления серверных файлов со статусом `UPDATING`.
- На странице сервера показываются код выхода и последнее событие agent.
- Agent автоматически проверяет требуемую Java для выбранной версии Minecraft и поддерживает настройки `KSYLIAN_JAVA_8`, `KSYLIAN_JAVA_17`, `KSYLIAN_JAVA_21`.
- При создании сервера можно задать минимальную и максимальную RAM; agent применяет их в Java-аргументах и `MemoryMax`.
- При создании сервера можно выбрать Java runtime, дополнительные JVM-аргументы и CPU limit.
- Agent проверяет свободное место перед созданием сервера и отдаёт предупреждения по ресурсам.
- Agent предупреждает о возможном зависании, если systemd-процесс запущен, но Minecraft status ping не отвечает.
- Во вкладке логов добавлены WebSocket-обновление, автопрокрутка с паузой, поиск, фильтры `INFO`, `WARN`, `ERROR`, `FATAL` и скачивание полного лога.
- Во вкладке логов добавлена RCON-командная строка с историей команд.
- Для новых серверов автоматически генерируются RCON-настройки и пароль.
- На странице сервера добавлена вкладка диагностики со списком файлов из `crash-reports`, stack trace, вероятной причиной, признаками конфликтующего мода, missing dependency, client-only мода и последними действиями agent.
- В milestone `Ksylian 0.2 — Stable Core` закрыты обязательная авторизация агента, запуск Minecraft не от root, исправленная установка Purpur, управление RAM, выбор Java, WebSocket-консоль, RCON и просмотр crash-reports.
- Добавлен базовый Minecraft proxy Ksylian.
- Подготовлена маршрутизация через wildcard-домен `<server>.<domain>`.
- Публичные адреса серверов показываются через agent при настроенном `KSYLIAN_PUBLIC_DOMAIN`.
- Онлайн и максимальное число игроков читаются через Minecraft status ping.

## Примечания по roadmap

Основной roadmap хранится в `docs/ROADMAP.md`.

Roadmap считаем источником правды. Если текущая реализация временно отличается от roadmap, это фиксируем в статусе как технический долг, а не как изменение направления.

## Следующая кандидатная задача

Milestone `Ksylian 0.2 — Stable Core`:

- Реализовать безопасный бэкап: `save-off`, `save-all flush`, архив, checksum/manifest и `save-on`.
- Добавить импорт существующего сервера.
- Проверить новые RCON/WebSocket сценарии на VPS.
